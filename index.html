Section 01

Introduction

What Shopify CLI is, why it exists, and what you can do with it as a theme developer.

Shopify CLI is an open-source command-line tool that streamlines Shopify theme and app development. Instead of manually uploading files through the browser editor, CLI gives you a live development server, file sync, Git-style workflows, and full theme lifecycle management from your terminal.

⚡
Hot Reloading
See changes instantly with shopify theme dev — no manual uploads.
↕
Theme Sync
Pull live theme files down and push your local changes back up seamlessly.
✓
Built-in Linting
Catch Liquid errors before they go live with shopify theme check.
⑂
Git-Friendly
Combine CLI with Git for branching, conflict resolution, and team workflows.

Why use Shopify CLI over the browser editor?

The browser theme editor is great for content changes, but for serious development it has real limitations. With CLI you get a proper code editor, version control, team collaboration, automated linting, and repeatable deployment pipelines. You work locally, keep Git history, and deploy with confidence.

💡
Tip
Shopify CLI v3 uses Node.js under the hood and is distributed via npm. It replaces the older Ruby-based CLI v2 with significantly faster performance and a cleaner API surface.

Section 02

Installation

Get Shopify CLI installed on macOS, Windows, or Linux in minutes.

Prerequisites

Before installing, make sure you have the following:

  • Node.js 18.20+ — Download from nodejs.org or use nvm
  • npm 8+ — Comes with Node. Check with npm -v
  • Git — Recommended for version control workflows
  • A Shopify Partner account — Free at partners.shopify.com

Install via npm (recommended)

Terminal 
# Install Shopify CLI and theme components globally
npm install -g @shopify/cli @shopify/theme

# Verify installation
shopify version

Install via Homebrew (macOS)

macOS Terminal 
brew tap shopify/shopify
brew install shopify-cli

Update CLI

Terminal 
npm update -g @shopify/cli @shopify/theme
⚠️
Windows Users
On Windows, use PowerShell or Windows Terminal. WSL2 (Windows Subsystem for Linux) is strongly recommended for the best experience. Avoid Git Bash for running Shopify CLI as it can cause path issues.

Section 03

Authentication

Connect Shopify CLI to your Shopify account securely.

Auth Commands 
# Log in (opens browser for OAuth)
shopify auth login

# Log in to a specific store
shopify auth login --store your-store.myshopify.com

# Check who you're logged in as
shopify whoami

# Log out
shopify auth logout

Authentication uses OAuth — the CLI opens your browser, you approve access, and a token is stored locally in your system keychain. You never need to paste API keys manually.

ℹ️
CI/CD Environments
For headless environments (GitHub Actions, CI pipelines), use the SHOPIFY_CLI_PARTNERS_TOKEN environment variable with a Shopify Partners API token instead of browser-based OAuth.

Section 04

Store Connection

Connect to a specific Shopify store and manage theme targets.

Most CLI commands accept a --store flag. You can also set a default store by running commands from a directory that already has a .shopify config, or by using environment variables.

Store Flags 
# Connect to a specific store inline
shopify theme dev --store your-store.myshopify.com

# Use a .env file for store config
# Create .env in your theme root:
SHOPIFY_FLAG_STORE=your-store.myshopify.com
SHOPIFY_CLI_THEME_TOKEN=shptka_xxxx

# CLI will auto-pick up .env variables
💡
Pro Tip — .env files
Add .env to your .gitignore to keep store credentials out of your repository. Use .env.example with placeholder values for teammates to reference.

Section 05

Dev Server

Start a local development server with hot reloading against a real Shopify store.

shopify theme dev is your primary command during active development. It watches local files, syncs changes to a development theme on your store, and provides a live preview URL.

shopify theme dev 
# Start dev server (creates a dev theme on store)
shopify theme dev

# Dev against a specific store
shopify theme dev --store your-store.myshopify.com

# Dev against an existing theme (by ID)
shopify theme dev --theme 123456789

# Open browser automatically
shopify theme dev --open

# Use a specific port for the local server
shopify theme dev --port 9292

# Skip pushing local files first (faster cold start)
shopify theme dev --nodelete

How it works

1
Upload to dev theme
CLI creates (or reuses) a hidden "development theme" on your store and pushes your local files to it.
2
Watch for changes
CLI watches your local file system. When you save a file, it syncs the change to the dev theme within milliseconds.
3
Live preview
Open the provided ?preview_theme_id=xxx URL — the browser auto-refreshes when files change.
4
Stop with Ctrl+C
The dev server stops and the development theme stays on your store for reuse next time.
💡
Dev themes are hidden
Development themes created by shopify theme dev are hidden from the Shopify admin theme list. They don't count against your theme limit and won't be accidentally published.

Section 06

Pull & Push

Sync theme files between your local machine and Shopify store.

theme pull — Download

shopify theme pull
# Pull current live theme
shopify theme pull

# Pull specific theme by ID
shopify theme pull --theme 123456789

# Pull to specific directory
shopify theme pull --path ./my-theme

# Pull only specific files
shopify theme pull --only templates/

theme push — Upload

shopify theme push
# Push to active theme (with confirmation)
shopify theme push

# Push as new unpublished theme
shopify theme push --unpublished

# Push to specific theme by ID
shopify theme push --theme 123456789

# Allow deleting remote files
shopify theme push --allow-live
🛑
Danger — Push overwrites remote
shopify theme push overwrites remote files with your local versions. If someone made changes in the Shopify editor while you were working locally, those changes will be lost. Always pull before pushing in collaborative environments.

Safe push workflow

Recommended workflow before push 
# 1. Stash current local changes
git stash

# 2. Pull latest from store
shopify theme pull

# 3. Commit pulled changes to Git
git add -A && git commit -m "sync: pull latest from store"

# 4. Pop your stashed work
git stash pop

# 5. Resolve any conflicts, then push
shopify theme push

Section 08

Theme Check

Lint and validate your Liquid, JSON, and assets to catch errors before they ship.

shopify theme check 
# Run full theme check
shopify theme check

# Auto-fix correctable issues
shopify theme check --auto-correct

# Check specific path only
shopify theme check --path ./sections

# Output as JSON (for CI pipelines)
shopify theme check --output json

# Fail with exit code on errors (for CI)
shopify theme check --fail-level error

Theme Check config (.theme-check.yml)

.theme-check.yml 
root: ./

# Disable specific checks
UnusedAssign:
  enabled: false

MissingTemplate:
  enabled: true

RemoteAsset:
  enabled: false
💡
Use in CI
Add shopify theme check --fail-level error to your GitHub Actions workflow to block merges when Liquid errors are present.

Section 09

.shopifyignore

Control which local files are excluded from push/pull operations.

Similar to .gitignore, a .shopifyignore file in your theme root tells Shopify CLI which files to exclude during sync operations. Useful for local development tooling files that shouldn't go to the store.

.shopifyignore 
# Local tooling — don't push to Shopify
*.test.liquid
node_modules/
.env
.env.*
.DS_Store
.vscode/
README.md
package.json
package-lock.json
*.config.js

# Staging-only sections (example)
sections/dev-banner.liquid

Section 12

Local Development

Setting up a solid local development environment and daily workflow.

Recommended project structure

Theme directory structure 
my-shopify-theme/
├── assets/          # CSS, JS, images, fonts
├── config/          # settings_schema.json, settings_data.json
├── layout/          # theme.liquid, password.liquid
├── locales/         # Translation JSON files
├── sections/        # Section .liquid files
├── snippets/        # Reusable Liquid snippets
├── templates/       # Page template files (.json or .liquid)
├── .shopifyignore   # Files to exclude from sync
├── .gitignore       # Files to exclude from Git
├── .env             # Local store config (never commit)
└── README.md        # Project docs

Daily development workflow

Typical dev session 
# Morning — sync latest changes from store
git checkout main
git pull origin main
shopify theme pull

# Create feature branch
git checkout -b feature/new-hero-section

# Start dev server
shopify theme dev

# Make changes, preview in browser…
# When done, commit and push to Git
git add -A
git commit -m "feat: add animated hero section"
git push origin feature/new-hero-section

# After review, merge to main and deploy
git checkout main
git merge feature/new-hero-section
shopify theme push --unpublished

Section 13

Git Workflow

Branching strategy, Git + Shopify CLI integration, and team collaboration patterns.

Recommended branching strategy

🌿 Branch naming

  • main — production-ready code
  • staging — pre-release testing
  • feature/* — new features
  • fix/* — bug fixes
  • hotfix/* — urgent production fixes
  • chore/* — refactors, tooling

📐 Git rules for themes

  • Never commit config/settings_data.json
  • Always pull before starting work
  • Use atomic, descriptive commits
  • Require PR reviews before merging main
  • Tag releases: v1.0.0
  • Keep .env out of Git

Recommended .gitignore for Shopify themes

.gitignore 
# Environment & secrets
.env
.env.*
!.env.example

# Shopify theme editor customizations (optional)
config/settings_data.json

# OS noise
.DS_Store
Thumbs.db
desktop.ini

# Editor
.vscode/settings.json
*.swp
*.swo

# Node
node_modules/
npm-debug.log*
⚠️
settings_data.json
Committing config/settings_data.json is controversial. It contains theme editor customizations (colors, text, section settings). Many teams exclude it to avoid content conflicts. Decide on a team policy and document it.

Section 14

Conflict Resolution

Handling merge conflicts, theme pull/push conflicts, and accidental overwrites.

Types of conflicts in Shopify theme work

⑂
Git merge conflicts
Two branches modified the same Liquid file. Standard Git resolution applies.
↕
Local vs remote conflicts
You edited locally while someone changed the same file in the Shopify editor.
🔄
settings_data conflicts
Theme customizer changes vs Git-tracked settings create JSON conflicts.
🗑
Accidental overwrite
A push wiped out changes made in the Shopify admin. Recovery needed.

Git merge conflict — what it looks like

📄 sections/hero.liquid — merge conflict
<<<<<<< HEAD (your branch)
<h1 class="hero__title hero__title--large">{{ section.settings.title }}</h1>
=======
<h1 class="hero__heading">{{ section.settings.heading }}</h1>
>>>>>>> main (their changes)
>>>>>>> end conflict

Resolving Git conflicts step-by-step

Git conflict resolution workflow 
# 1. See what's conflicted
git status
# Unmerged paths: sections/hero.liquid

# 2. Open conflicted file in editor, resolve markers:
#    <<<<<<< HEAD  → your changes
#    =======        → divider
#    >>>>>>> main   → incoming changes
#    Delete markers, keep the correct code

# 3. Stage resolved file
git add sections/hero.liquid

# 4. Continue merge or commit
git merge --continue
# or
git commit -m "merge: resolve hero section conflict"

# 5. Push resolved code to store
shopify theme push

Recovering from accidental theme push overwrite

Recovery workflow 
# Option A — Restore from Git history
git log --oneline -10
# Find the last good commit hash, e.g. abc1234
git checkout abc1234 -- sections/hero.liquid
git commit -m "revert: restore hero.liquid from abc1234"
shopify theme push --only sections/hero.liquid

# Option B — Full rollback to previous Git commit
git revert HEAD
shopify theme push

# Option C — Use Shopify admin theme history
# Admin → Online Store → Themes → ⋯ → Revert to previous version

Git stash for safe context switching

git stash workflow 
# Urgent hotfix needed mid-feature? Stash work:
git stash push -m "WIP: hero animation feature"

# Do the hotfix
git checkout hotfix/fix-nav-mobile
# … fix things …
shopify theme push
git checkout feature/hero-animation

# Restore stashed work
git stash pop

# View all stashes
git stash list

# Apply specific stash
git stash apply stash@{2}

Section 15

Deployment

Safe deployment patterns, staging environments, and GitHub Actions CI/CD.

Manual deployment workflow

Recommended deploy process 
# 1. Lint before deployment
shopify theme check --fail-level error

# 2. Push as new UNPUBLISHED theme (safe preview)
shopify theme push --unpublished --store your-store.myshopify.com
# CLI outputs the new theme ID

# 3. Preview the unpublished theme, get approval
shopify theme open --theme NEW_THEME_ID

# 4. Publish when ready (go live)
shopify theme publish --theme NEW_THEME_ID

# 5. Tag the release in Git
git tag -a v1.4.2 -m "Release: new hero + nav refactor"
git push origin --tags

GitHub Actions CI/CD

.github/workflows/deploy.yml 
name: Deploy Theme

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Shopify CLI
        run: npm install -g @shopify/cli @shopify/theme

      - name: Lint theme
        run: shopify theme check --fail-level error

      - name: Deploy to store
        run: shopify theme push --unpublished --allow-live
        env:
          SHOPIFY_CLI_THEME_TOKEN: ${{ secrets.SHOPIFY_CLI_THEME_TOKEN }}
          SHOPIFY_FLAG_STORE: ${{ secrets.SHOPIFY_STORE }}
💡
Store the token securely
Add SHOPIFY_CLI_THEME_TOKEN and SHOPIFY_STORE as GitHub Actions secrets in your repository settings. Generate the token from Shopify Partners → Apps → API keys.

Section 16

Collaboration

Team workflows, roles, and best practices for multi-developer Shopify projects.

Team workflow overview

In a team environment, the key challenge is keeping local development, Git history, and the live Shopify theme in sync without stepping on each other's work.

📐
Golden rule
One person owns the live theme push. Others use --unpublished for review. No direct pushes to the live theme without review — enforce this with protected branches.

Theme environments pattern

Multi-environment workflow 
# Developer A — works on feature branch
git checkout -b feature/product-gallery
shopify theme dev
# Previews in their own dev theme (hidden)

# Developer A — creates a review theme for client
shopify theme push --unpublished --name "Review: Product Gallery"
shopify theme share
# Share the preview link in PR/Slack

# Lead Developer — merges PR and deploys
git checkout main
git merge feature/product-gallery
shopify theme push --unpublished
# After sign-off:
shopify theme publish --theme THEME_ID

Section 17

Debugging & Troubleshooting

Common errors, CLI issues, and how to fix them.

The CLI isn't on your PATH. Try:

Fix
npm list -g @shopify/cli
# If not installed:
npm install -g @shopify/cli @shopify/theme
# Verify:
which shopify

Never use sudo npm install -g. Instead, fix npm permissions:

Fix
# Use nvm to manage Node versions
nvm use --lts
npm install -g @shopify/cli @shopify/theme

File watchers can fail on large projects or when hot-reloading stalls. Try:

Fix
# Stop server (Ctrl+C), then restart
shopify theme dev --verbose
# On Linux, increase file watcher limit:
echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
Fix
shopify auth logout
shopify auth login
# If using .env token, regenerate it in Partners dashboard

CLI requires explicit permission to push to the published live theme.

Fix
# Use --allow-live flag to confirm intent
shopify theme push --allow-live
# Or push as unpublished (safer)
shopify theme push --unpublished

Section 18

Best Practices

Proven patterns from real Shopify theme development teams.

✅ Do

  • Always pull before pushing
  • Use unpublished pushes for review
  • Lint with theme check before deploy
  • Keep .env out of Git
  • Use atomic Git commits
  • Tag releases with semver
  • Use .shopifyignore
  • Test in multiple browsers
  • Use dev themes, not the live theme
  • Document your branching strategy

❌ Avoid

  • Pushing directly to live theme
  • Committing .env or secrets
  • Editing live theme in browser during dev
  • sudo npm install -g
  • Ignoring merge conflicts
  • Pushing without theme check
  • Working without Git branches
  • Skipping the PR review step
  • Large untested pushes to production
  • Overriding team merge decisions alone

VS Code Setup

Recommended extensions and settings for Shopify theme development in VS Code.

Recommended extensions

💧
Shopify Liquid
Official extension — syntax highlighting, hover docs, and autocomplete for Liquid.
✨
Prettier
Auto-format Liquid, JS, CSS, and JSON on save. Install the Liquid Prettier plugin.
🔍
Theme Check
Inline Liquid linting directly in the editor. Shows errors without running CLI manually.
⑂
GitLens
Enhanced Git integration — file blame, history, compare, and team insights.
🔐
DotENV
Syntax highlighting for .env files. Simple but useful.
📏
ESLint
Catch JavaScript quality issues in your theme's assets/ scripts.

Recommended .vscode/settings.json

.vscode/settings.json 
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "[liquid]": {
    "editor.defaultFormatter": "Shopify.theme-check-vscode"

  "editor.tabSize": 2,
  "files.associations": {
    "*.liquid": "liquid"
  },
  "editor.wordWrap": "on",
